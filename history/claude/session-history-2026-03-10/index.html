    📋 REPORTING PROTOCOL (Established 2026-02-19)¶

    Write reports PER TASK, not per session. Document while context is fresh.

    How To Report:¶

    # After completing a task, run:
~/scripts/nexus-report.sh <project-path> "<title>" "<detailed body>"

# Examples:
~/scripts/nexus-report.sh ~/git/medusa-proxy "Added ExcludeNodes" "What/Why/How/Files..."
~/scripts/nexus-report.sh ~/scripts "Created nexus-foo.sh" "What/Why/How/Files..."

    Report Body Template:¶

    • What: One sentence - what was built/changed
    • Why: The problem or need
    • How: Technical details, approach taken
    • Files: List of files created/modified
    • Dependencies: What it needs to run (if any)

    Where Reports Live:¶

    • Project reports: <project>/docs/action_report.md (per-task entries)
    • Central hub: ~/claude/reports/ (symlinks to all project reports)
    • Master report: ~/claude/action_report.md (one-line summaries with links)
    • Initialize a new project: ~/scripts/nexus-report.sh --init <project-path>
    • List all reports: ~/scripts/nexus-report.sh --list

    Rules:¶

    1. Report IMMEDIATELY after completing each task
    2. Every project gets docs/action_report.md - init with --init if missing
    3. Help files (*.help.md) live next to their script in ~/scripts/
    4. NEVER pile everything into one giant session report at the end
